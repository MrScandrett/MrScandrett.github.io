About ClassroomOS

Mr. Scandrett's ClassroomOS is the public front door for class. It brings together lesson launches, student showcase pages, classroom app links, downloads, and special labs in one GitHub Pages site.

Static on GitHub Pages Teacher-curated publishing Lessons + Apps + Showcase

What The Site Does Now

The homepage works as a classroom launchpad. From there, students can open STEAM lessons, browse published projects, launch classroom tools, and jump into special spaces like Music Lab, the Recipe Book, and Class Downloads.

The public site is designed to be simple at the top level and deeper where needed: lessons open interactive pages, showcase entries open project detail pages, and classroom applications open the tools students already use.

Home Launchpad

STEAM Lessons

The lesson library holds interactive AI and STEAM activities, including labs, simulations, and historical computing lessons built for classroom use.

Student Showcase

Student work is published as individual browser-ready projects. The showcase supports discovery, filtering, thumbnails, and dedicated project pages.

Applications

The applications page is a launch panel for classroom tools such as Scratch, Tinkercad, Clever, Code.org, and GitHub, with class-specific Tinkercad entry links.

Downloads + Resources

Class Downloads and the Recipe Book provide reusable files, project prompts, templates, and classroom build ideas without sending students through extra menus.

Special Labs

Some pages go beyond simple navigation. Music Lab, simulations, and project viewers give students live, browser-based tools they can use during class.

How Student Work Reaches The Showcase

1. Prepare

Student files are collected in a classroom project folder and checked so they can run cleanly in a browser.

2. Build

Projects are packaged into standalone app folders for the site. That includes web projects and supported 3D model viewers.

3. Curate

Thumbnails, titles, descriptions, and metadata are reviewed before a project appears in the public showcase.

4. Publish

The final project opens directly from this site so students, teachers, and families can view it without extra setup.

Public-Site Rules

  • Projects are reviewed before they are linked on the public site.
  • Personal information should not appear in public descriptions, screenshots, or outbound links.
  • Published work should open clearly in the browser and be classroom-appropriate.
  • Showcase entries are meant to spotlight finished or share-ready student work, not every draft.

Why This Exists

This site keeps class entry points consistent. Students know where to start, teachers have a clean place to direct traffic, and published work has a home that feels intentional instead of scattered.

The public site does not need a login to browse. Publishing decisions stay teacher-controlled.
