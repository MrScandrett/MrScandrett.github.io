How to Put Your HTML Project Online

Fast, clean, and beginner-friendly GitHub Pages workflow.

Share link for LMS/class page: https://YOUR-USERNAME.github.io/YOUR-REPO/

1Create a GitHub Account

Go to github.com and make a free account.

Screenshot placeholder: GitHub sign up screen

2Create a New Repository

Choose a name like march-showcase. Set visibility to Public.

Screenshot placeholder: New repository form

3Upload Project Files

Make sure index.html is in the top folder (root), not nested inside another folder.

Screenshot placeholder: Repo file list with index.html at root

4Turn on GitHub Pages

Open Settings → Pages. Source should be main branch and root folder.

Screenshot placeholder: Pages settings panel

5Get Your Live Link

Your website link will look like: https://YOUR-USERNAME.github.io/YOUR-REPO/

Your tutorial link will look like: https://YOUR-USERNAME.github.io/YOUR-REPO/tutorial/

Screenshot placeholder: Published site open in browser

Checklist

  • index.html is in the root folder
  • All images/scripts/styles are uploaded
  • No local file paths like C:\ or file://
  • Repo is public and Pages is enabled

Common Mistakes

  • Missing index.html: add it to project root.
  • Wrong folder depth: avoid nesting project folders inside each other.
  • Broken asset paths: use relative paths like ./images/pic.png.
